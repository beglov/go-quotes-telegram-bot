# План разработки бота цитат

1. Подготовка структуры проекта и зависимостей

- Инициализировать `go.mod`, добавить `cmd/bot/main.go`, директории `internal/quotes`, `internal/subscribers`, `internal/scheduler`, `internal/bot`.
- Подключить `github.com/go-telegram-bot-api/telegram-bot-api/v5` и стандартные пакеты.

2. Клиент Forismatic и доменная логика цитат

- Создать `internal/quotes/forismatic_client.go` с типом клиента, выполняющим HTTP-запросы к `api.forismatic.com` (JSON, lang=ru).
- Реализовать повторную попытку при ошибках, обработку отсутствующих авторов, возврат структуры цитаты.
- Добавить модульный тест с `httptest.Server` на успешный ответ и недостающего автора.

3. Хранилище подписчиков в JSON-файле

- Реализовать `internal/subscribers/store.go` с интерфейсом для подписки/отписки, чтения/записи в `data/subscribers.json`.
- Обеспечить потокобезопасность (мьютекс), автосоздание файла, проверку дубликатов.
- Добавить тесты на базовые операции и устойчивость к пустому/поврежденному файлу.

4. Планировщик ежедневной рассылки

- Создать `internal/scheduler/daily.go`, рассчитывающий время до следующего запуска заданного часа/минуты и вызывающий callback в отдельной горутине.
- Протестировать вычисление интервала.

5. Telegram-бот и команды

- Реализовать `internal/bot/service.go`, инъектируя телеграм-API, клиент цитат и стор подписчиков.
- Обработать команды `/start`, `/stop`, `/quote`, логирование, отправку сообщений с fallback при ошибках.
- Подключить ежедневный планировщик для рассылки всем подписчикам.

6. Точка входа и конфигурация

- В `cmd/bot/main.go` загрузить конфиг (переменные окружения для токена и времени), создать зависимости, запустить обработку обновлений и планировщик.
- Добавить README с инструкциями по запуску и настройке.
